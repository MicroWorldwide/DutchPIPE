  197.      * Call this method from action methods, for example actionFoo(). When the
  198.      * user performs an action but the action fails, there are two ways for the
  199.      * method implementing the action to communicate the failure to the user.
  200.      *
  201.      * <ol>

  204.      * example:<br><br>

  215.      * failure setup is used when different objects can have the same actions

  221.      * </code>
  222.      * <br>
  223.      * The action system will continue looking for other ways to perform
  224.      * the action. If it doesn't find any, the previously set failure message
  225.      * is communicated to the user. Otherwise, the next method implementing
  226.      * the action takes over. The action system will continue looking for
  227.      * object/methods implementing the action, until TRUE is returned or no
  228.      * more implementations are found. In that case, the last set action
