  225. /**
  226.  * Enable internationalization/localization support with dynamic gettext?
  227.  *
  228.  * This has a speed penalty. Set to TRUE to enable or FALSE to disable.
  229.  *
  230.  * Different languages are supported using GNU gettext, see
  231.  * {@link http://www.php.net/gettext}. Because PHP is an interpreted language,
  232.  * gettext slows things down as calls are dynamic (although it's quite fast).
  233.  * There are some tools around to make static PHP translations, but this has not
  234.  * been explored yet. Also, it is not yet possible to have multiple users use
  235.  * different languages on one site. It's in one language.
  236.  *
  237.  * Currently, standard English ("en") and Dutch ("nl_NL")  are part of the
  238.  * standard distribution. Also, all texts to translate are in only one file
  239.  * called messages.po (this will change in the future).
  240.  *

  242.  * locale/en/LC_MESSAGES/messages.po and please mail your results to
  243.  * contributors@dutchpipe.org. As I did with the Dutch translation, you'll
  244.  * probably run into a couple of problems getting it exactly right, because
  245.  * of presumptions made by the English language system. Please let us know
  246.  * so this can be fixed.
  247.  *
  248.  * @see        DPSERVER_GETTEXT_LOCALE_PATH, DPSERVER_GETTEXT_DOMAIN,
